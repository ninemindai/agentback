Actor runtime: addressable, serialized state

invoke(type, id, command) → one turn at a time per identity, over a swappable ActorRuntime port

Addressing

  • • invoke(type, id, command) — the id is the address
  • • cart/ada and cart/grace are independent state
  • • REST controllers and MCP tools are callers
  • • actors do not become endpoints by themselves

One turn at a time

  • • serialized per identity; concurrent across identities
  • • load → receive(clone) → validate → commit
  • • throw or schema-invalid → rollback, nothing commits
  • • a committed requestId replays its result

A port, not a framework

  • • ActorDefinition is the seam — no decorators/DI below it
  • • in-memory reference adapter + Redis (lease + atomic Lua)
  • • every adapter passes the same conformance suite
  • • experimental spike; Durable Objects is the next gate
