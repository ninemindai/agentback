messaging — Architecture Map

Durable, distributed messaging substrate for the AgentBack runtime · ports + swappable adapters · bus under the engine

Two ports, two axes

  • • JobQueue = durable job/worker (BullMQ)
  • • EventBus = pub/sub fan-out (Streams)
  • • Scheduler = cron, as repeatable jobs
  • • WebSocket = a subscriber, not a bus

Bus under the engine

  • • Triggers/spawns enqueue jobs
  • • Workers pull → drive same engine
  • • Crash-durable detached executions
  • • Engine stays the multi-step brain

Pluggable, Redis default

  • • Ports have zero backend deps
  • • messaging-bullmq is the durable Redis adapter
  • • in-memory adapter for dev/test
  • • SQS/NATS later, same contract
