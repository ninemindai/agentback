AgentBack Actor Programming Model

Decorated DI services discovered through Component.services, compiled to a runtime contract, and serialized per actor identity.

01 · Programming topology
02 · One invoke, success and rollback

Authoring rule

  • • Implement Actor and decorate command methods.
  • • Contribute actor classes via Component.services.
  • • Address state by type + ID.
  • • Registry compiles metadata to ActorDefinition.

Concurrency rule

  • • One active turn per identity.
  • • Different identities run concurrently.
  • • No cross-actor transaction or reentrancy.

Safety boundary

  • • State rollback does not undo side effects.
  • • Use outbox or idempotent effect handlers.
  • • Durable adapters need atomic state + inbox.

Beyond commands

  • • @actorQuery: read-only, lease-free, concurrent.
  • • ref(Class, id) typed proxy / @injectActor accessor.
  • • Commands may emit events → per-identity event log.
