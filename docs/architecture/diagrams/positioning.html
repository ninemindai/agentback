AgentBack — Positioning

One Zod schema becomes every boundary — a REST/HTTP API for developers and an MCP surface for agents — served on any host.

The drift it removes

  • • One contract, today copied into 4–5 places
  • • Zod, OpenAPI, an MCP manifest, a client, a skill file
  • • Edit one, miss another → they diverge
  • • AgentBack derives them all from the schema

Developers — the REST API

  • • REST routes with Zod request/response validation
  • • OpenAPI 3.1, auto-emitted from the same schema
  • • A typed client — z.infer, no codegen
  • • Swagger UI at /explorer

Agents — the MCP surface

  • • MCP tools from the same @tool schema
  • • Machine-readable docs: /llms.txt
  • • A derived agent skill (SKILL.md) — emerging
  • • Recoverable errors: codes, issues, hints

Runs anywhere

  • • One fetchHandler() — fetch(Request)
  • • Node (native), Express, Fastify, Hono
  • • Bun, Deno, Cloudflare Workers
  • • REST and MCP on every host
