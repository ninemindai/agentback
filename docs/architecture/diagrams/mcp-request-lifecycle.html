AgentBack — /mcp Request Lifecycle

A POST flows ①→⑦ through the group-sorted cascade into the route; an OPTIONS preflight short-circuits at ①. Companion to middleware-chain.html.

Resolved order (POST)

  • • Chain is the 1st handler; cascade is group-sorted: cors → parseBody → middleware
  • • Each calls next(); on cascade end, toExpressMiddleware calls Express next() → router
  • • /mcp route (mounted by installMcpHttp) is the destination behind the chain
  • • ⑤ the route's own express.json() no-ops (req._body set in ②)

Short-circuit exits

  • • OPTIONS preflight → cors ends 204 at ①, no next() (cheapest exit)
  • • rate limit exceeded → 429 at ③, before the router
  • • auth guard rejects → 401 at ④, transport never built
  • • A non-preflight POST never short-circuits at ① — cors just stamps headers

Layering

  • • Edge concerns (cors, body, rate-limit) live in the chain — uniform across all routes
  • • Identity (④ req.auth/scopes) lives on the /mcp route — session/scope-aware
  • • Different layers, no collision: that's why the chain-first + per-session changes compose
  • • bodyParser:false → ② skipped, ⑤ becomes the real parser; /mcp still works
