# v0.5.1

Patch — hotfix for a production crash introduced in 0.5.0.

## 🐛 Fix

**`ERR_REQUIRE_ESM` crash on Vercel / non-require-of-ESM runtimes
(`@agentback/rest`).** `rest.server` lazy-loaded the middleware-chain helpers via
`createRequire('@agentback/middleware')`, but `@agentback/middleware` is
ESM-only — `require()`-ing it throws `ERR_REQUIRE_ESM` on runtimes that don't
support require-of-ESM (e.g. Vercel serverless functions). Every Express-host
app (`RestApplication` + the `install*` UIs / `mcp-http`) crashed at `start()`
with `FUNCTION_INVOCATION_FAILED`. Node ≥ 22.13 permits require-of-ESM, so it
passed local dev and only surfaced in production.

The helpers are now **statically imported** from `@agentback/middleware` (a
neutral, Express-free package already on the static graph for
`EXPRESS_SERVICE_KEY`); only the CJS optional peers (`express`/`cors`) still use
the `createRequire` loaders. A source-graph regression test now fails if
`rest.server` ever `require()`s an `@agentback/*` package again.

No API changes; no migration. **Upgrade from 0.5.0 is recommended for anyone
deploying the Express host to a serverless platform.** Edge (`EdgeRestApplication`)
apps were unaffected (they never load the Express path).
