# v0.5.2

Patch — follow-up to the 0.5.1 serverless fix.

## 🐛 Fix

**`agentback deploy vercel` generated a function that crashed with `Cannot find
module 'cors'`.** Since 0.5.0, `@agentback/rest` loads `express`/`cors` lazily
via `createRequire` (to keep edge builds Express-free). Vercel's bundler
(`node-file-trace`) follows only static imports, so it dropped `express`/`cors`
from the deployed function — the Express-host entry then couldn't load them at
runtime. The generated Vercel entry (`@agentback/cli`'s `generate-entry`) now
**static-imports `express` + `cors`**, pulling them (and their transitive deps)
into the trace. Reproduced and fixed on the live demo.

No API changes. Relevant only to apps deployed via `agentback deploy vercel`
(or any Express host on a serverless platform — see the new "Deploying the
Express host to a serverless platform" section in
[docs/guides/deploy-to-edge.md](../guides/deploy-to-edge.md)). Your app must
still declare `express` + `cors` as dependencies (they are optional peers of
`@agentback/rest`).
