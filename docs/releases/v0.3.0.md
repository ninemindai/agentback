# v0.3.0

Lockstep release. Headline: **first-class file uploads/downloads**, a
**front-mounted middleware chain** with configurable body parsing, and
**session/user-scoped MCP** over Streamable HTTP. Two new packages
(`@agentback/files`, `@agentback/files-s3`).

## ✨ Highlights

### First-class file uploads & downloads

No more multer escape hatch. Declare a `fileField()` in a route's `body:`
schema and one declaration drives the whole chain:

- **Upload** — an auto-mounted, per-route multipart parser **streams** each file
  to the bound `FileStore` under a **server-generated UUID key** (no
  client-controlled paths), delivering validated `UploadedFile` handles in the
  slot-0 input bundle.
- **OpenAPI** — the body emits as `multipart/form-data` with `format: binary`,
  so upload routes stay visible in `/openapi.json` and `/llms.txt`.
- **Download** — `return fileResponse(stream, …)` / `fileDownload(retrieved)`;
  the server streams it (Content-Type / Content-Disposition) instead of
  JSON-encoding.
- **Security baked in** — disallowed MIME types rejected **pre-stream** (`415`,
  no orphaned object), a **default 25 MiB cap** so uploads are never unbounded,
  sanitized download filenames, and a demonstrated owner-scoping pattern.

### New packages

- **`@agentback/files`** — transport-agnostic `FileStore` port
  (`put`/`get`/`exists`/`delete` + optional presigned hooks) with
  **`InMemoryFileStore`** and **`FsFileStore`** (local disk) adapters, plus a
  shared `@agentback/files/testing` conformance suite.
- **`@agentback/files-s3`** — S3 adapter (AWS SDK v3 + `lib-storage`
  streaming), implementing the presigned hooks for direct-to-S3 flows.

### Middleware chain front-mount + configurable body parsing

- The LB-style middleware chain now mounts as the **first** Express handler in
  the `RestServer` constructor (matching upstream LB4's `ExpressServer`), so
  `app.middleware()` / CORS / rate-limiting correctly front **every** route —
  including `install*` surfaces (`/mcp`, `/console`, `/explorer`) that
  previously **shadowed** the chain.
- **CORS and body parsing are now group-sorted chain entries**
  (`cors → parseBody → middleware`) via the new `RestMiddlewareGroups`.
- **`RestServerConfig.bodyParser`** — JSON-only by default, `false` for none,
  or any of `json` / `urlencoded` / `text` / `raw`, so a server can accept
  media types beyond `application/json`.

### Session / user-scoped MCP over Streamable HTTP

`@agentback/mcp-http` gains a `perSession` binder (+ `appContext`): each MCP
session can get its own DI-scoped server, enabling per-user tool surfaces over
the Streamable HTTP transport.

## 🔧 Fixes

- `urlencoded` body parsing now defaults `{extended: true}`, silencing the
  Express 4 deprecation; completed the body-parser test matrix.
- Multipart hardening: pre-stream MIME rejection (no orphaned stored objects)
  and a default size cap.

## 🧭 New public API

- `@agentback/files`: `FileStore`, `FileNotFoundError`, `InMemoryFileStore`,
  `FsFileStore`, `FILE_STORE`, `@agentback/files/testing`
- `@agentback/files-s3`: `S3FileStore`
- `@agentback/openapi`: `fileField()`, `fileFieldsOf()`, `UploadedFile`
- `@agentback/rest`: `fileResponse()`, `fileDownload()`,
  `RestBindings.HTTP_REQUEST` / `HTTP_RESPONSE`, `RestMiddlewareGroups`,
  `RestServerConfig.bodyParser`

## ⚠️ Behavior changes (0.x, API still settling)

- **Middleware ordering** — the chain now fronts `install*` routes. If you
  relied on the old behavior where `app.middleware()` did **not** run for
  `/mcp`, that's now fixed (it does). CORS/body parsing keep their effective
  positions.
- **`@agentback/rest` now depends on `multer`** (direct dependency; a candidate
  to optionalize as a peer dep in a future release).
- A new per-request binding (`RestBindings.HTTP_REQUEST` / `HTTP_RESPONSE`) is
  bound on every request.

## 📚 Docs & examples

- New architecture diagrams: the middleware chain and the `/mcp` request
