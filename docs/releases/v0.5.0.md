# v0.5.0

Lockstep release. Headline: **a real `RestApplication` now runs on Cloudflare
Workers** (and any Web-standard edge runtime), end-to-end — bundle-clean *and*
runtime-clean — plus a new `agentback deploy` CLI (Vercel + Cloudflare) and the
`EdgeRestApplication` / `@agentback/middleware` split that lets edge apps drop
the `express` install entirely.

> **Breaking changes** (below) in `@agentback/files`, `@agentback/context`,
> `@agentback/rest`, and `@agentback/express` — the minor bump from `0.4.x` they
> require.

## ✨ Highlights

### Full edge-readiness — `RestApplication` on Cloudflare Workers

A real app (controllers + `/openapi.json` + `/llms.txt`) deploys via
`agentback deploy cloudflare` and serves on `workers.dev`. Two layers of work:

- **Bundle-clean** — severed Node-only modules (`node:fs`/`node:fs/promises`/
  `node:net`) from the static graph of a fetch-only app: `@agentback/express`
  is reached via subpaths (not the barrel, which re-exports the Express
  server), `multer` is lazy-loaded, and `FsFileStore`/`fromDisk` no longer ride
  the barrel into edge bundles.
- **Runtime-clean** — eliminated global-scope randomness at module load
  (ID generation is call-time only) and the Express-mounting that `start()` did
  in native mode; hardened the `.env` loader's Node-detection guard against
  Cloudflare's `nodejs_compat` (which fakes `process.versions.node`).

**Edge apps must construct with `rest: {listener: 'native'}`** — see
[docs/guides/deploy-to-edge.md](../guides/deploy-to-edge.md). In native mode the
runtime-neutral `fetchHandler()` is the single router; `start()` mounts nothing
on Express. The placeholder-vs-schema and Express-coupled-route guardrails now
fire at `start()` on the native path too (parity with Express).

## ⚠️ Breaking changes

### `@agentback/files`: `FsFileStore` moved to the `/fs` subpath

`FsFileStore` pulls `node:fs`, which is edge-hostile, so it is no longer
re-exported from the package barrel (importers that only want the `FileStore`
port / `FILE_STORE` key now stay edge-clean).

```diff
- import {FsFileStore} from '@agentback/files';
+ import {FsFileStore} from '@agentback/files/fs';
```

`InMemoryFileStore`, the `FileStore` port, and `FILE_STORE` are unchanged on the
barrel. (`@agentback/files-s3`'s `S3FileStore` is unaffected.)

### `@agentback/context`: removed the deprecated `uuid()` helper + `UUID_PATTERN`

Both were `@deprecated` and unused internally. Replace with:

```diff
- import {uuid} from '@agentback/context';
- const id = uuid();
+ const id = crypto.randomUUID();        // or generateUniqueId() for internal ids
```

`generateUniqueId()` is unchanged in name but is now backed by
`@agentback/common`'s `generateIdSync` (nanoid) instead of `hyperid`; its output
format changed (no monotonic `-<counter>` suffix). It remains `@internal`.

### `@agentback/rest`: `express`/`cors`/`multer` are now optional peer deps

The neutral middleware machinery moved to the new **`@agentback/middleware`**
package, and `@agentback/express` became the optional Express **host**. As a
result `@agentback/rest` no longer hard-depends on `express`/`cors`/`multer` —
they are **optional `peerDependencies`**, so they are NOT auto-installed.

**Migration — if you run the Node/Express host** (`RestApplication`, a.k.a.
`ExpressRestApplication`, or any app using `app.middleware`/`app.expressMiddleware`,
the `install*` UIs like `@agentback/console`/`rest-explorer`, or `mcp-http` on
the default listener), **add `express` + `cors` to your app's own dependencies**
(and `multer` if you use `fileField()` uploads). In `0.4.x` these arrived
transitively via `@agentback/rest → @agentback/express`; in `0.5.0` you declare
them yourself:

```diff
  "dependencies": {
    "@agentback/rest": "^0.5.0",
+   "express": "^5.2.1",
+   "cors": "~2.8.6"
+   // "multer": "^2.2.0"   // only if you use fileField() uploads
  }
```

If you forget, the Express host throws an actionable error at `start()`
("the Express host requires the optional peer dependencies 'express' and 'cors'
… or use `EdgeRestApplication`"). **Edge / fetch-only apps need none of this** —
see below.

### New: `EdgeRestApplication` (fetch/edge host) + `@agentback/middleware`

Host is now an explicit class choice. `RestApplication` (= `ExpressRestApplication`)
is the Node/Express host; **`EdgeRestApplication`** is pinned to
`listener: 'native'`, serves through `fetchHandler()`, and its dependency
closure contains **no `express`/`cors`** — a Cloudflare Workers / Bun / Deno app
installs neither. It deliberately omits `app.middleware`/`app.expressMiddleware`
(Express-only); use `app.webMiddleware`. Supported surface: `@api` routes,
`/openapi.json`, `/llms.txt`, MCP-over-fetch (the `install*` UIs remain
Express-host-only for now).

## 🔧 Notable internal changes

- `@agentback/context` drops the `hyperid` and `uuid` dependencies.
- Route placeholder-vs-schema validation extracted to a shared module so the
  Express and fetch/native paths enforce it identically.
